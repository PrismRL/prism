class Collision¶

Keeps track of different move types or “layers” in the game for use in collision detection.

MOVETYPES: table<string, integer>¶

Collision movetypes used for bitmasking. Each bit represents a movetype, or collision layer. Each movetypes represents a different collision category.

registeredMovetypes: table<string, integer>¶

Stores registered movetypes names mapped to their bitmask values.

movetypeNames: table<integer, string>¶

Stores bitmask values mapped to their registered movetypes names.

staticmethod registerMovetype(name: string, movetype: string)¶

Registers a user-defined name for a collision movetypes. Prevents duplicate registrations and invalid movetypes.

Parameters:

  • name (string) – The name to associate with the movetypes.

  • movetype (string) – The movetypes key from Collision.movetypes.

staticmethod getMovetypeByName(name: string) → value: integer | nil¶

Retrieves the bitmask value associated with a registered movetypes name.

Parameters:

name (string) – The registered movetypes name.

Returns:

value (integer | nil) – The bitmask value, or nil if not found.

staticmethod getMovetypeName(bitmask: integer) → name: string | nil¶

Retrieves the registered movetypes name associated with a bitmask value.

Parameters:

bitmask (integer) – The bitmask value.

Returns:

name (string | nil) – The movetypes name, or nil if not found.

staticmethod createBitmaskFromMovetypes(movetypesNames: string[]) → The: integer¶

Converts a list of collision movetypes names into a combined bitmask.

Parameters:

movetypesNames (string[]) – A list of movetypes names to combine.

Returns:

The (integer) – combined bitmask.

staticmethod checkBitmaskOverlap(bitmaskA: integer, bitmaskB: integer) → True: boolean¶

Checks if two bitmasks have any overlapping collision movetypes.

Parameters:

  • bitmaskA (integer) – The first bitmask.

  • bitmaskB (integer) – The second bitmask.

Returns:

True (boolean) – if there is an overlap, false otherwise.
