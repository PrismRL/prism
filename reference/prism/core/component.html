class Component : Object¶

The Component class represents a component that can be attached to actors or cells. Components are used to add functionality to actors. For instance, the Moveable component allows an actor to move around the map. Components are essentially data storage that can also grant actions.

requirements: Component[]¶

(static) A list of components (prototypes) the entity must have before this one can be applied.

owner: Entity¶

The entity this component is composing. This is set by Entity when a component is added or removed.

getRequirements(self: Component) → ...: Component¶

Returns a list of components (prototypes) the entity must have before this one can be applied. Override this to provide requirements, and it will get called to populate the list.

checkRequirements(self: Component, entity: Entity) → meetsRequirements: boolean, Component?¶

Checks whether an actor has the required components to attach this component.

Parameters:

entity (Entity) – The actor to check the requirements against.

Returns:

  • meetsRequirements (boolean) – True if the entity meets all requirements, false otherwise.

  • _2? (Component) – The first component found missing from the entity if requirements aren’t met.

__new(self: Object, ...: any)¶

The default constructor for the class. Subclasses should override this.
